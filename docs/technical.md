Описание технического задания для Cloud провайдера «Хакатон IaaS МТС»

Техническое задание: 
Смоделировать проект облачной платформы (IaaS) для Cloud провайдера.

Описание: 
Разработать концепт упрощённой облачной платформы (IaaS) для управления виртуальными ресурсами, демонстрирующий основные принципы работы cloud провайдера.
Решение должно показывать:
- мультитенантность;
- логическую изоляцию клиентов;
- подход к распределнию и контролю ресурсов.
В качестве основы для облачной платформы может использоваться гипервизор Proxmox, однако его применение не является обязательным и может быть представлено на концептуальном уровне, в виде mock-интеграции или архитектурной схемы.

Задание:
Разработать облачную платформу (IaaS), включающую:
- веб-интерфейс для клиентов — личный кабинет, где клиент может просматривать свои виртуальные ресурсы (ВМ или аналог), управлять их состоянием (создание, запуск, остановка — концептуально);
- простую панель администратора — где администратор может управлять клиентами (tenants), назначать лимиты ресурсов (CPU, RAM, диск, количество виртуальных ресурсов), отслеживать общее использование инфраструктуры.
Изоляция виртуальных ресурсов и сетей между клиентами может быть показана логически и концептуально.
Допускается частичная реализация, mock-интеграция и использование схем вместо реальной инфраструктуры.

Возможные варианты визуализации и интерфейса:
    1. Визуализация клиентской панели (список ВМ, статус, управление);
    2. Визуализация админ-панели (клиенты, квоты, использование ресурсов);
    3. Концептуальные схемы сетевой изоляции и распределения ресурсов;
    4. Простые диаграммы или графики по использованию ресурсов.

Варианты направлений применения (демонстрация функционала): 
    1. Мультитенантная облачный сервис;
    2. Контроль и распределение ресурсов между клиентами;
    3. Логическая изоляция и безопасность;
    4. Упрощённое управление виртуальной инфраструктурой;
    5. Администрирование ресурсов на уровне провайдера;
    6. Любые дополнительные сценарии использования по выбору команды.

Идеи по расширению возможностей платформы приветствуются.

Критерии оценки проектов для прохождения в финал и выбора команды-победителя: 
    1. Идея и ценность решения
Насколько понятно, для какой аудитории предназначен облачный сервис, какую задачу он решает и в чём заключается ценность предложенного подхода;
    2. Пользовательский опыт и сценарии использования
Качество проработки пользовательского пути клиента и администратора, логика взаимодействия с платформой, удобство и понятность сценариев;
    3. Креативность и вариативность подхода
Оригинальность идей, нестандартные сценарии использования облачного сервиса, дополнительные функции и расширения платформы;
    4. Проработка концепта облачной платформы
Понимание и корректное отражение принципов работы облачных решений: мультитенантность, логическая изоляция клиентов, распределение и контроль ресурсов.
Оценивается концептуальная логика, а не глубина технической реализации;
    5. Визуализация и наглядность решения
Качество и понятность визуальных материалов: интерфейсов, схем, диаграмм и других демонстрационных элементов;
    6. Целостность и завершённость проекта
Насколько решение выглядит логичным, целостным и готовым к обсуждению и дальнейшему развитию, даже при частичной реализации;
    7. Презентация и командная работа
Качество и структура презентации, участие всех членов команды, умение аргументировать принятые решения и работа в команде в течение хакатона.
